# RPG 表格模板参考

以下是游戏数据库的预定义表格模板。使用 `python scripts/db.py create-table` 创建表格，引用对应的列头。

---

## 模板速查

| 模板键 | 中文名 | 列头 | 使用场景 |
|--------|--------|------|----------|
| `global-state` | 全局状态 | 角色名, 当前地点, 当前时间, 上轮时间, 经过时间, 备注 | 追踪时间线、地点变化 |
| `protagonist-info` | 主角信息 | 角色名, 性别/年龄, 外貌特征, 当前衣着, 职业/身份, 备注 | 主角核心身份档案 |
| `character-stats` | 角色属性 | 角色名, 属性名, 当前值, 最大值, 备注 | 跟踪 HP、MP、力量等数值属性 |
| `inventory` | 背包物品 | 物品名称, 数量, 类别, 归属, 描述, 备注 | 管理多人背包/物品栏 |
| `quests` | 任务日志 | 任务名称, 任务类型, 状态, 发布者, 详细描述, 当前进度, 任务时限, 奖励, 惩罚 | 追踪任务完整生命周期 |
| `npc-relations` | 角色关系 | 姓名, 性别/年龄, 外貌特征, 当前衣着, 身份/职业, 好感度, 是否在场, 背景故事, 关键经历, 人际关系, 简介, 别称, 备注 | 深度 NPC 档案与关系网 |
| `protagonist-skills` | 主角技能 | 技能名称, 技能类型, 等级/阶段, 消耗, 效果描述 | 管理主角已习得的技能 |
| `equipment` | 装备栏 | 装备名, 部位, 属性加成, 耐久度, 归属, 描述 | 追踪多人装备状态 |
| `memo` | 备忘录 | 标题, 分类, 归档状态, 记录日期, 记录内容, 涉及对象, 关键要素 | 线索/成就/约定追踪 |
| `factions` | 势力表 | 名称, 描述, 领袖, 关系, 位置 | 管理组织与势力 |
| `locations` | 地点表 | 地点名称, 所属区域, 地点描述, 地点现状 | 重要地点档案 |
| `chronicle` | 纪要表 | 时间跨度, 地点, 纪要, 概览, 编码索引 | 每轮剧情事件日志 |

---

## 操作权限体系

每个表格都有操作规则，分为四类：

| 操作 | 含义 | 何时允许 |
|------|------|----------|
| **初始化** | 首次填充数据 | 游戏开始时，根据设定和剧情插入初始数据 |
| **插入** | 添加新行 | 新物品获得、新 NPC 登场、新任务触发、新地点发现、新势力出现 |
| **更新** | 修改已有数据 | 属性变化、物品数量增减、任务进度推进、NPC 状态变化、好感度改变 |
| **删除** | 移除整行 | 物品完全消耗/丢弃、势力覆灭、地点彻底消失。大部分表禁止删除 |

**核心原则：**
- **数据不丢** —— 已完成的任务改为"已完成"状态，不删除；已离场 NPC 标记"否"，不删除
- **归档锁定** —— 备忘录的"已归档"和"已废除"状态是不可逆的，一旦设置禁止任何更新
- **追加优于修改** —— 纪要表只追加新行，不修改已有记录
- **唯一条目约束** —— 全局状态和主角信息表仅有一行，禁止新增或删除

## 1. 全局状态 (global-state)

追踪游戏世界的时间线、主角当前位置。此表 **有且仅有一行**。

**创建命令：**
```bash
python scripts/db.py create-table global-state --headers "角色名,当前地点,当前时间,上轮时间,经过时间,备注"
```

**列说明：**

| 列 | 规则 |
|----|------|
| **角色名** | 主角名称 |
| **当前地点** | 主角当前所在的具体场景名称。从大到小描述（如"灰炉镇→猎荒公会大厅"） |
| **当前时间** | 游戏世界的当前时间。格式：`x年x月x日 HH:MM`。初始化时必须根据世界观设定一个明确的日期时间，不能用未知数代替。具体年月叫法依据当前世界观 |
| **上轮时间** | 上一轮交互结束时的时间 |
| **经过时间** | 根据当前与上轮时间计算得出的文本描述（如"约1小时"、"几分钟"、"0分"） |

**操作规则：**

| 权限 | 规则 |
|------|------|
| **初始化** | 插入唯一一条记录，包含主角初始位置和当前世界时间 |
| **更新** | 主角切换地点时更新"当前地点"；每轮交互结束时更新时间字段 |
| **插入** | **禁止操作** —— 此表仅一行 |
| **删除** | **禁止操作** —— 此表仅一行 |

**插入示例：**
```bash
python scripts/db.py insert-row global-state --data '{"角色名":"艾伦","当前地点":"冒险者公会大厅","当前时间":"星历1250年霜月3日 14:30","上轮时间":"","经过时间":"0分","备注":""}'
```

## 2. 主角信息 (protagonist-info)

记录主角的核心身份信息。此表 **有且仅有一行**。

**创建命令：**
```bash
python scripts/db.py create-table protagonist-info --headers "角色名,性别/年龄,外貌特征,当前衣着,职业/身份,备注"
```

**列说明：**

| 列 | 规则 |
|----|------|
| **角色名** | 主角的名字 |
| **性别/年龄** | 生理性别和年龄，格式如 `男/22`、`女/19` |
| **外貌特征** | ⚠️ 必须包含：发型、发色、身高、身材、肤色、五官。**严禁包含：** 神态描写（如"嘴角勾起一抹冷笑"）、服装、姿势或临时状态。**低频更新**：除非发生重大剧情改变（断肢、疤痕等永久性变化），否则不更新 |
| **当前衣着** | 当前服装搭配和装饰。**低频更新**：换装或衣物破损时更新。服装顺序建议：上身→下身→鞋→配饰 |
| **职业/身份** | 主角在社会中的主要角色或称号 |
| **备注** | 补充说明 |

**操作规则：**

| 权限 | 规则 |
|------|------|
| **初始化** | 游戏初始化时插入主角唯一记录 |
| **更新** | 外貌特征仅重大改变时更新；衣着在换装/破损时更新 |
| **插入** | **禁止操作** |
| **删除** | **禁止操作** |

**插入示例：**
```bash
python scripts/db.py insert-row protagonist-info --data '{"角色名":"艾伦","性别/年龄":"男/22","外貌特征":"深棕色短发，身高178cm，体型中等，小麦色皮肤，五官轮廓分明","当前衣着":"深蓝旅行斗篷、灰色布制上衣、黑色长裤、皮革长靴","职业/身份":"冒险者","备注":""}'
```

## 3. 角色属性 (character-stats)

跟踪角色的核心数值属性，支持多角色各自记录。每个属性一行。

**创建命令：**
```bash
python scripts/db.py create-table character-stats --headers "角色名,属性名,当前值,最大值,备注"
```

**列说明：**

| 列 | 规则 |
|----|------|
| **角色名** | 属性所属角色名称 |
| **属性名** | 属性名称（生命值/法力值/力量/敏捷/智力/感知/星辉值 等） |
| **当前值** | 当前数值 |
| **最大值** | 最大上限 |
| **备注** | 属性来源或特效说明（如"+2 来自装备"、"升级获得"） |

**操作规则：**

| 权限 | 规则 |
|------|------|
| **初始化** | 根据角色设定，逐行插入初始属性 |
| **更新** | 战斗中受伤/治疗、升级加点、装备加成变化时更新 |
| **插入** | 获得新属性体系（如觉醒新能力）时添加 |
| **删除** | 属性被永久剥夺时删除 |

**插入示例：**
```bash
python scripts/db.py insert-row character-stats --data '{"角色名":"艾伦","属性名":"生命值","当前值":"100","最大值":"100","备注":"基础生命"}'
```

## 4. 背包物品 (inventory)

管理角色携带的物品、消耗品和道具。支持多人背包，通过「归属」列区分。

**创建命令：**
```bash
python scripts/db.py create-table inventory --headers "物品名称,数量,类别,归属,描述,备注"
```

**列说明：**

| 列 | 规则 |
|----|------|
| **物品名称** | 物品的名字 |
| **数量** | 持有数量，必须 > 0。数量归零时删除该行 |
| **类别** | 武器 / 消耗品 / 货币 / 任务物品 / 材料 / 杂物 等 |
| **归属** | 物品所有人的姓名。区分多人背包 |
| **描述** | 物品功能或背景描述 |
| **备注** | 当前状态（如"已遗失"、"已损坏"、"已使用"）。正常状态留空 |

**操作规则：**

| 权限 | 规则 |
|------|------|
| **初始化** | 根据设定添加主角和 NPC 的初始物品 |
| **更新** | 获得已有物品→数量+N；消耗→数量-N；物品状态变化→更新备注 |
| **插入** | 获得新物品时添加 |
| **删除** | 数量归零时删除；物品被丢弃/摧毁时删除 |

**插入示例：**
```bash
python scripts/db.py insert-row inventory --data '{"物品名称":"治疗药水","数量":"3","类别":"消耗品","归属":"艾伦","描述":"恢复50点生命值","备注":""}'
```

## 5. 任务日志 (quests)

追踪所有任务的完整生命周期。

**创建命令：**
```bash
python scripts/db.py create-table quests --headers "任务名称,任务类型,状态,发布者,详细描述,当前进度,任务时限,奖励,惩罚"
```

**列说明：**

| 列 | 规则 |
|----|------|
| **任务名称** | 任务标题 |
| **任务类型** | 主线任务 / 支线任务 |
| **状态** | 未开始 / 进行中 / 已完成 / 已失败 |
| **发布者** | 发布该任务的角色或势力。自发任务填"（自发）" |
| **详细描述** | 任务的目标和要求 |
| **当前进度** | 对完成度的简要描述。初始填"无" |
| **任务时限** | 剩余时间限制。无限制填"无" |
| **奖励** | 完成可获得的奖励。未知填"未知" |
| **惩罚** | 失败的后果。无惩罚填"无" |

**操作规则：**

| 权限 | 规则 |
|------|------|
| **初始化** | 根据剧情添加初始主线任务 |
| **更新** | 取得关键进展→更新"当前进度"；完成→状态"已完成"；失败/过期→状态"已失败"。**不要删除已完成的任务** |
| **插入** | 接取/触发新任务时添加 |
| **删除** | 仅当任务彻底废除且与剧情无关时删除（极少使用） |

**插入示例：**
```bash
python scripts/db.py insert-row quests --data '{"任务名称":"消灭哥布林","任务类型":"支线任务","状态":"进行中","发布者":"村长","详细描述":"清除森林东侧洞穴中的哥布林","当前进度":"已找到洞穴入口","任务时限":"3天内","奖励":"50金币","惩罚":"声望-10"}'
```

## 6. 角色关系 (npc-relations)

深度记录 NPC 档案与关系网。每个重要 NPC 一行。

**创建命令：**
```bash
python scripts/db.py create-table npc-relations --headers "姓名,性别/年龄,外貌特征,当前衣着,身份/职业,好感度,是否在场,背景故事,关键经历,人际关系,简介,别称,备注"
```

**列说明：**

| 列 | 规则 |
|----|------|
| **姓名** | NPC 名字 |
| **性别/年龄** | 格式如 `女/25`、`男/58` |
| **外貌特征** | ⚠️ 必须包含：发型、发色、身高、身材、肤色、五官。**严禁包含：** 神态描写、服装、姿势或临时状态。仅在身体发生永久性改变时更新 |
| **当前衣着** | 当前服装搭配。换装/破损时更新 |
| **身份/职业** | NPC 的社会角色或职业 |
| **好感度** | 数值范围 -100（仇敌）到 +100（至交）。反映 NPC 对主角的单方面态度倾向 |
| **是否在场** | 是 / 否。标记 NPC 是否在主角当前场景中 |
| **背景故事** | ⚠️ 格式：按 `1、2、3` 数字列表，上限 5 条。每条 ≤ 30 token。总 token ≤ 150。已知则填，信息未知填"未知"。记录遇到主角**之前**的背景 |
| **关键经历** | ⚠️ 格式：按 `1、2、3` 数字列表，上限 10 条。每条 ≤ 30 token。**总 token ≤ 300**。接近上限时合并最早且内容最相近的 2-3 条。记录遇到主角**之后**的关键事件 |
| **人际关系** | 格式 `{角色名}:{关系词}`，多组用分号分隔。每组仅限一个角色。关系词：恋人/挚友/师徒/主仆/仇人/队友/父子/母子/父女/母女/兄妹/姐弟/亲人/亲戚。**仅记录对剧情有重大影响的核心关系**，弱关系（点头之交）不录 |
| **简介** | 一句话客观概括（≤ 30 token） |
| **别称** | 简称和外号，英文逗号分隔（如"老铁,铁叔"） |
| **备注** | 补充信息。若角色死亡，在此标注"已死亡" |

**操作规则：**

| 权限 | 规则 |
|------|------|
| **初始化** | 游戏开始时，为当前在场的重要 NPC 逐行插入 |
| **更新** | 状态变化（离场/死亡）、好感度变化、关系变化、关键经历新增时更新。⚠️ 每次更新关键经历后必须检查总 token ≤ 300！ |
| **插入** | 剧情中有新的重要 NPC 登场时添加 |
| **删除** | **禁止删除** —— NPC 离场标记"否"，死亡在姓名旁备注"（已死亡）" |

**⚠️ Token 压缩策略：** 关键经历列总 token 接近 300 时，将最早且内容最相近的 2-3 条合并为一条（≤ 30 token），保持总 token ≤ 300。

**插入示例：**
```bash
python scripts/db.py insert-row npc-relations --data '{"姓名":"艾琳","性别/年龄":"女/25","外貌特征":"金色长发，身高165cm，苗条，白皙，碧绿眼眸","当前衣着":"白色公会制服、蓝色短裙、棕色长靴","身份/职业":"冒险者公会接待员","好感度":"75","是否在场":"是","背景故事":"1.曾是冒险者，因伤退居后方 2.在公会工作已三年","关键经历":"1.为主角登记了冒险者身份","人际关系":"主角:提供帮助","简介":"公会银发的温柔接待员","别称":"小艾","备注":""}'
```

## 9. 备忘录 (memo)

记录主角获得的线索、成就、约定等。**不是流水账**，只在事件闭环结束后汇总记录。

**创建命令：**
```bash
python scripts/db.py create-table memo --headers "标题,分类,归档状态,记录日期,记录内容,涉及对象,关键要素"
```

**列说明：**

| 列 | 规则 |
|----|------|
| **标题** | 备忘录标题，内容的概括 |
| **分类** | 必须从以下三类中选择：秘密线索类 / 大事件类 / 约定承诺类 |
| **归档状态** | 未归档 / 已归档 / 已废除。默认插入时为"未归档"。⚠️ **一旦设为已归档或已废除，本行所有字段均不可再更新** |
| **记录日期** | 游戏内日期，格式 `x年x月x日 HH:MM` |
| **记录内容** | 客观、详细的事件记录。⚠️ **不升华、不评价**。总 token ≤ 300 |
| **涉及对象** | 备忘录涉及的角色名 |
| **关键要素** | 核心要素名词或短语，最多 2 个，用于触发检索 |

**分类说明：**

| 分类 | 适用场景 | 更新规则 |
|------|----------|----------|
| **秘密线索类** | 获得关键情报、发现隐藏信息、解开谜题 | 可更新：记录获取事实本身 + 后续查证情况 |
| **大事件类** | 完成重要任务、经历重大事件、角色成长突破、获得成就 | 可更新：记录事件发展过程 |
| **约定承诺类** | 与他人达成约定、许下承诺、签订契约 | 可更新：承诺建立 + 后续履行情况 |

**操作规则：**

| 权限 | 规则 |
|------|------|
| **初始化** | 游戏开始时如有已存在的重要备忘录则插入 |
| **更新** | ⚠️ 仅当归档状态为"未归档"时才允许更新。根据分类规则更新对应的"记录内容"和"归档状态"。**任何情况下均不可更新"分类"字段** |
| **插入** | ⚠️ 只有完整事件闭环结束才可添加。对连续性挑战/流程，**禁止在中间环节添加**，必须等待流程彻底结束汇总为一条 |
| **删除** | **禁止删除** |

**⚠️ 颗粒度控制：**
1. **严禁流水账** —— 连续多阶段事件（连续闯关、持续战斗、系列交涉）不可在每个小步骤后单独记录
2. **事件闭环** —— 必须等到系列事件彻底结束产生最终结果后，汇总为一条高密度备忘录
3. **结果导向** —— 只记录改变现状的决定性结果，不记录过程细节

**插入示例：**
```bash
python scripts/db.py insert-row memo --data '{"标题":"发现哥布林巢穴","分类":"秘密线索类","归档状态":"未归档","记录日期":"星历1250年霜月3日 10:00","记录内容":"在森林东侧发现了哥布林的洞穴入口，门口有哨兵巡逻","涉及对象":"艾伦","关键要素":"哥布林巢穴"}'
```

## 12. 纪要表 (chronicle)

每轮游戏交互的剧情日志。**每轮结束后追加一条**。

**创建命令：**
```bash
python scripts/db.py create-table chronicle --headers "时间跨度,地点,纪要,概览,编码索引"
```

**列说明：**

| 列 | 规则 |
|----|------|
| **时间跨度** | 本轮事件发生的时间范围，格式 `开始时间 ~ 结束时间` |
| **地点** | 本轮事件发生的地点，从大到小描述 |
| **纪要** | ⚠️ **如实连贯概括，主次分明，无结案陈词。** 以旁观者身份按时间顺序客观记录核心事件。**抓主放次**：聚焦核心事件推进，跳过无关紧要的动作细节。**严禁**在末尾做任何总结升华。总字数 ≥ 100 字 |
| **概览** | 30 字以内，2-3 句话概括本轮要点 |
| **编码索引** | 格式 `AMXXXX`，从 `AM0001` 递增，唯一标识。用于快速检索历史轮次 |

**操作规则：**

| 权限 | 规则 |
|------|------|
| **初始化** | 游戏开始时插入一条初始化记录 |
| **更新** | **禁止操作** —— 纪要不可修改 |
| **插入** | 每轮交互结束后追加一条新记录 |
| **删除** | **禁止操作** —— 历史记录不可删除 |

**插入示例：**
```bash
python scripts/db.py insert-row chronicle --data '{"时间跨度":"星历1250年霜月3日 08:00 ~ 08:30","地点":"主角的家→冒险者公会","纪要":"艾伦从家中出发前往冒险者公会。在公会大厅遇到了接待员艾琳，完成了冒险者身份登记。","概览":"主角登记成为冒险者，结识接待员艾琳","编码索引":"AM0001"}'
```

## Token 管理指南

在 RPG 场景中，上下文窗口是有限的。以下表格可能消耗大量 Token，需要主动管理：

| 表格 | 风险 | 管理策略 |
|------|------|----------|
| **npc-relations.关键经历** | ⚠️ 高 | 上限 300 token，接近上限时合并压缩 |
| **npc-relations.背景故事** | 🔸 中 | 上限 150 token，已知后不追加 |
| **memo.记录内容** | 🔸 中 | 上限 300 token，事件闭环后才记录 |
| **chronicle.纪要** | ⚠️ 高 | 仅当前轮次相关的前 20 条注入上下文 |

**Token 估算方法：** 中文字符约 1 字符 ≈ 0.5 token，英文约 1 词 ≈ 1 token。粗略估算：`总 Token ≈ 中文字符数 × 0.5 + 英文词数`。
